# Signal Atlas — AI 信息星图

## 产品定位

用数据视角看 AI 信息生态的交互式可视化产品。别人看内容，你看内容背后的数据。

**一句话**：155+ AI 信源、数千篇文章的质量评分和话题关系，做成一张可交互的星空图谱。

## 范围

只聚焦 AI 相关内容：
- overseas（46 个信源）：海外 AI newsletter、博客、YouTube
- wechat-ai（109 个信源）：国内 AI/Tech 公众号

不包含投资频道。

## 核心数据优势

- 2 个月+ 持续评分数据（7 维度 LLM 评分，0-100 分）
- 155+ AI 信源，跨中外
- 每日更新，成本 ~$0.06/天
- 关键词提取 + 信号雷达 + 反馈闭环
- 同一话题的中外信息差数据

## 视觉风格

**参考**：NASA Eyes on Asteroids（https://eyes.nasa.gov/apps/asteroids）
- 深空背景 + 辉光效果
- 多层级缩放：远看全景，近看细节
- 进来就动，无加载页
- 暗色主题，科技感

## 核心设计：力导向图谱（Obsidian Graph 启发）

**核心理念**：关系从内容中自然涌现，不是预设分类。

### 数据组织：方向 → 主题 → 文章

```
方向（星区）                主题（星座）              文章（星点）
──────────              ──────────            ──────────
自然涌现，不硬编码   →   共享关键词的文章聚成簇  →  每篇文章是一颗星
                        密集区域自动标注主题名
```

### 自下而上的涌现机制

1. 每篇文章有关键词（评分系统已提取）
2. 两篇文章共享关键词 → 产生一条边
3. 力导向布局：共享关键词多的文章自然靠近
4. 密集区域自动标注为"主题"（如 Agent、RAG、多模态）
5. 相近的主题簇 → 形成"方向"

### 星空隐喻映射

| 现实概念 | 星空隐喻 |
|---------|---------|
| 文章 | 星点（亮度=评分，大小=影响力） |
| 主题（Agent、RAG...） | 星座（自然聚类 + 连线 + 标签） |
| 方向（应用/基建/产业/前沿） | 星区（空间上的大区域） |
| 信源 | 文章属性（点击星点看"来自 Simon Willison"） |
| 今日新文章 | 星光闪烁 |
| 话题降温 | 星座变暗、散开 |
| 中外信息差 | 海外星 vs 国内星的颜色区分 |

### 交互层级

| 缩放距离 | 看到什么 |
|---------|---------|
| 远（全景） | 几个大的方向区域，主题星座轮廓 |
| 中 | 星座名称，星点分布密度 |
| 近 | 具体文章标题，评分，信源名 |
| 点击 | 文章详情卡片（摘要、评分、原文链接） |

## 5 个核心页面

1. **首页/星空图谱** — 力导向布局的文章星空，缩放探索，主题星座自然涌现
2. **信源排行** — 信源质量排名、活跃度、信噪比，信源画像
3. **话题趋势** — 关键词热度曲线，话题生命周期，星座的演变
4. **中外对比** — 同一话题在海外和国内的时间差、关注度差异
5. **文章存档** — 所有被评分过的文章，可筛选可搜索（标题+评分+摘要+原文链接）

## 技术方案

- **框架**：Next.js 静态导出
- **图谱渲染**：Three.js + React Three Fiber（辉光 bloom 效果）
- **布局引擎**：d3-force（力导向模拟，关键词共现 → 边权重）
- **数据**：预计算 JSON，每天简报跑完自动更新
- **部署**：makinote.cn 子路径或子域名
- **无需后端**

## 合规

- 数据来源于公开 RSS，属于衍生数据分析
- 不展示原文全文，只放标题+评分+摘要+链接
- 页面底部声明
